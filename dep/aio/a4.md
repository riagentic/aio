  THE LOOP

  action → reduce(state, action) → new state + effects
                                        ↓           ↓
                                    UI updates   effects run
                                                     ↓
                                              (may send more actions)

  You define state. Things happen (actions). State changes (reduce).
  Work gets done (effects). UI reflects state. aio persists and syncs.

  YOUR APP — 7 files in src/

  state.ts      What's my data?          type + initial values
  actions.ts    What can happen?          action catalog
  effects.ts    What work needs doing?    effect catalog
  reduce.ts     How does data change?     (state, action) → { state, effects }
  execute.ts    How does work get done?   (app, effect) → void
  App.tsx       What does user see?       React component
  app.ts        Start.                    aio.run(initialState, config)

  CORE API                                   import { ... } from 'aio'

  actions({ Name: (params) => payload })     define action catalog
  effects({ Name: (params) => payload })     define effect catalog
  draft(state, d => { mutate d })            Immer helper for reduce
  useAio<S>()                                → { state, send }  (browser)
  connectCli<S>(url, opts?)                  → CliApp           (terminal)
  send(A.name(payload))                      dispatch action from UI
  page(current, { route: Component })        simple routing
  aio.run(initialState, { reduce, execute }) start everything

  PRODUCTION — compile:<shell>:<topology>

  Two axes: shell (what renders) × topology (local or remote).
  Local = self-contained, 127.0.0.1. Remote = exposed 0.0.0.0 or client-only.

                    local (default)              remote
  ┌────────────┬─────────────────────────┬──────────────────────────┐
  │ browser    │ compile:browser         │ compile:browser:remote   │
  │            │ binary + system browser │ exposed server + systemd │
  ├────────────┼─────────────────────────┼──────────────────────────┤
  │ electron   │ compile:electron        │ compile:electron:remote  │
  │            │ AppImage/zip/zip¹       │ client AppImage, no Deno │
  ├────────────┼─────────────────────────┼──────────────────────────┤
  │ cli        │ compile:cli             │ compile:cli:remote       │

  PERSISTENCE — automatic, zero config

  State → Deno.Kv          survives restart, deep-merged with schema
    persistMode:'multi'    one KV key per state prop — no 65KB limit
  Arrays → SQLite           structured data, queryable, indexed
    .whereOr / .upsert     OR filters, insert-or-replace
    .all/.where opts        orderBy / limit / offset
  Android → localStorage   standalone, no server

  NETWORK

  Default: localhost only. --expose: binds 0.0.0.0 + auto-HTTPS + token auth.
  Self-signed cert auto-generated (.aio-tls/). BYO: --cert=cert.pem --key=key.pem
  Multi-user: users: { token: { name, role } } in config.
  getUIState(state, user?) filters what each user sees.
